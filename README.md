# PiDAR
An autonomous robot that builds a real-time 2D map of an unknown room using LiDAR and SLAM, with live visualization and path tracking, and even a custom development board for all of its locomotive needs!

The Pi would be programmed in Python because it is better for data processing, visualizations, and (potentially) machine learning algorithms.

## Organization and Flow
This project is basically divided into a few tiny subproblems that each component is designed to tackle. 
1) The LiDAR (at the top) is meant to collect data by sweeping the polar grid and recording ToF data when obstacles are detected (it knows this because its signal would bounce back). The LiDAR then feeds this real-time data to a Raspberry Pi 4 Model B.
2) The Pi, on the layer below, processes the data and allows for stuff like visualizations and algorithms. It's basically the brain of the robot. It takes an input from the layer above, processes it, and then handles an appropriate response such as sending a signal to the kineticBoard or responding to a request on its webserver.
3) The kineticBoard, at the very bottom, connects to the +12V LiPo and handles all of the low-level stuff with the motors. It connects to the encoders and reads the pulses generated by its fluctuating magnetic field, and also drives a response from its built-in TB6612FNG motor driver to the motor terminals themselves, allowing the robot to move forward, backward, spin, etc.

It's kind of like the OSI & TCP/IP model because the abstraction being lowered and lowered with every layer.
